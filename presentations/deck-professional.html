Bot Design

Three Isolated Trading Engines

Bot Location Trigger Loop State
Entry Bot app.py — _tracking_loop() User submits symbol + params via UI 1 s (day) instant (swing) In-memory globals + DB
Exit Bot bot/trading_bot.py — TradingBot "Start Bot" in UI 5 s (day) 60 s (swing) TradingBot instance attrs + DB
BrownBot app.py — scanner + exit loops "Start" in BrownBot tab (per user) Scanner 30 s Exits 2 s BrownBotSession per user + DB

Bots share zero state. BrownBot positions go to brown_positions table; Entry/Exit Bot positions go to positions table. Modifying one bot cannot break another.

BrownBot — Core Design

Per-User Session Isolation

Each user who starts BrownBot gets an independent BrownBotSession dataclass. No global mutable state is shared between users.

@dataclass class BrownBotSession: active_positions: dict # symbol → position dict stats: dict # entry/exit counts per type logs: list # last 100 activity entries playbook_cache: dict # symbol → AI playbook result playbook_pending: set # symbols being fetched in background playbook_failed: set # symbols to skip on retry lock: threading.Lock # guards concurrent access # factory — creates if not exists, never returns stale state session = _get_brown_session(user_id)

Claude AI Integration

AI Playbook — Deferred Entry Pattern

To avoid blocking the 30 s scanner loop, AI evaluation is async. Entry is deliberately deferred by one full scan cycle.

# Scan cycle N — first encounter if symbol not in session.playbook_cache: if symbol not in session.playbook_pending: threading.Thread(target=_fetch_and_cache_playbook, ...).start() session.playbook_pending.add(symbol) continue # skip this cycle — wait for AI result # Scan cycle N+1 — AI result is cached result = session.playbook_cache[symbol] if result['bias'] == 'Short' and result['confidence'] == 'High': continue # skip — AI says historically bearish # Override stop/target with AI-suggested percentages stop_pct = result.get('stop_pct', config['day_stop_pct']) target_pct = result.get('target_pct', config['day_target_pct'])
Model: claude-sonnet-4-6 Cached per symbol per session Failed symbols added to skip-set
Exit Logic

Exit Priority Order

Checks run every 2 s per position. Conditions are evaluated in strict priority order — first match wins.

PriorityConditionTypeTrigger
1 Profit target Both price ≥ entry × (1 + target_pct%)
2 Stop loss Both price ≤ stop price (or breakeven stop)
3 EOD flatten Day only ET time ≥ day_eod_exit_time (default 15:55)
4 Max hold days Swing only hold_days ≥ swing_max_hold_days
5 Earnings protection Swing only earnings within swing_earnings_exit_days

Breakeven stop: once price reaches breakeven_trigger_pct% toward target, stop is moved to entry price. Sets _at_breakeven = True flag shown as "BE" badge in UI.

bot/risk_manager.py

Risk Manager — Circuit Breaker Design

Instantiated from brown_bot_config at bot start. Stateless — all decisions are computed from live inputs on each call.

def can_enter(self, symbol, position_type, active_positions, unrealized_pnl=0.0): realized = self._get_daily_pnl() # DB query → brown_positions today total = realized + unrealized_pnl # Circuit breaker — both components count if total <= self.max_daily_loss: return False, f"Daily loss limit hit: realized {realized:.2f} unrealized {unrealized_pnl:.2f}" # Slot cap — exit-pending positions excluded from count active_day = sum(1 for p in positions.values() if p['position_type'] in ('day', 'brown_day') and not p.get('_exit_pending')) if position_type == 'day' and active_day >= self.max_concurrent_day: return False, "Max day positions reached" return True, ""
Positive loss input normalised to negative automatically 22 unit tests covering all branches
Data Pipeline

Gap-Up Detection — Single Source of Truth

gap_up_detector.py → get_gap_up_stocks_for_frontend() is the only fetch path. BrownBot, the frontend, and the monitor loop all read from one source.

Session WindowPrimary SourceCache TTLStale Guard
pre_market / closed yfinance day_gainers 300 s —
open 09:30–09:31 ET yfinance (forced) 30 s Stale-window guard — Alpaca movers resets at 09:30, returns prior day until ~09:32
open ≥ 09:32 ET Alpaca movers + most-actives + universe scan (10K symbols) 30 s —
after_hours DB snapshot + universe scan 300 s —

Universe scan is non-blocking: 10–20 s for ~10K symbols. Cache miss spawns a UniversePrewarm daemon thread. Current request returns without universe data; next call (≥30 s) gets a hit.

Critical migration rule: CREATE TABLE IF NOT EXISTS must precede ALTER TABLE ADD COLUMN for the same table in _migrate_schema(). ALTER silently fails on a fresh DB if the table doesn't exist yet.

Error table
